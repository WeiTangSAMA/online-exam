# 在线考试系统使用手册

本手册面向系统使用者，说明如何登录、参加考试、维护题库、组卷发布、查看成绩和分析数据。系统基于浏览器使用，无需安装客户端。

## 1. 访问系统

### 1.1 启动地址

在浏览器地址栏打开：

```text
http://localhost:5173
```

如果打不开页面，请确认前端服务已启动：

```bash
cd online-exam-frontend
npm run dev
```

如果页面能打开但接口请求失败，请确认后端服务已启动：

```bash
cd online-exam-backend
mvn spring-boot:run
```

后端默认地址为 `http://localhost:8080`。

### 1.2 登录账号

系统内置以下测试账号，密码均为 `123456`。

| 用户名 | 角色 | 适用场景 |
| --- | --- | --- |
| `admin` | 管理员 | 完整管理权限，适合演示全部功能 |
| `teacher` | 教师 | 维护题库、组卷发布、查看成绩和统计 |
| `student1` | 学生 | 参加考试、查看个人成绩 |
| `student2` | 学生 | 同上 |
| `student3` | 学生 | 同上 |

登录页底部提供测试账号快捷按钮，点击后会自动填入用户名和密码。

### 1.3 注册账号

1. 在登录页切换到“注册”。
2. 填写用户名、密码、姓名。
3. 选择角色：学生或教师。
4. 点击“创建账号”。
5. 注册成功后系统会回到登录页，可使用新账号登录。

注意：管理员账号不支持前台注册，需要通过数据库或已有管理员手动创建。

### 1.4 退出登录

点击右上角头像区域，在下拉菜单中选择“退出登录”。退出后会返回登录页。

## 2. 工作台

登录后默认进入工作台。工作台会根据当前角色展示不同内容。

### 管理员 / 教师工作台

可查看：

- 题目总数
- 试卷总数
- 已发布试卷数量
- 成绩记录数量
- 题库管理、试卷管理、成绩管理、数据分析快捷入口

建议使用顺序：先维护题库，再创建试卷，发布后让学生参加考试，最后查看成绩和统计。

### 学生工作台

可查看：

- 可参加考试数量
- 已考次数
- 在线考试快捷入口
- 我的成绩快捷入口

学生一般从“在线考试”进入考试，从“成绩管理”查看结果。

## 3. 管理员 / 教师操作指南

以下功能需要使用 `admin` 或 `teacher` 登录。

### 3.1 题库管理

菜单路径：左侧导航 -> 题库管理

#### 3.1.1 查看题目

题库页面以表格展示题目，通常包含题型、分类、题干、答案、分值、创建时间等信息。可以通过题型、分类、关键字进行筛选。

#### 3.1.2 新增题目

1. 点击“新增题目”。
2. 选择题型：单选题、多选题或判断题。
3. 选择题目分类。
4. 填写题干内容。
5. 设置选项：
   - 单选题和多选题需要填写选项。
   - 判断题不需要选项。
6. 填写正确答案：
   - 单选题填写一个选项，如 `A`。
   - 多选题填写多个选项，如 `A,B,C`。
   - 判断题使用 `T` 表示正确，`F` 表示错误。
7. 设置分值。
8. 可选填写答案解析。
9. 点击“确定”保存。

#### 3.1.3 编辑题目

在题目列表中点击“编辑”，修改内容后点击“确定”。如果题目已绑定到试卷，修改分值后相关试卷总分会自动同步更新。

#### 3.1.4 删除题目

在题目列表中点击“删除”，确认后删除。

注意：已有学生答题记录的题目不允许删除，以避免历史成绩详情丢失。未作答题目删除时，会同步清理试卷题目关联并重新计算试卷总分。

#### 3.1.5 分类管理

进入分类管理后可以新增或删除题目分类。删除分类时，相关题目的分类会被置空，题目本身不会被删除。

### 3.2 试卷管理

菜单路径：左侧导航 -> 试卷管理

#### 3.2.1 创建试卷

1. 点击“新增试卷”。
2. 填写试卷标题。
3. 设置考试时长，单位为分钟。
4. 设置及格分。
5. 选择状态：
   - 草稿：暂不对学生开放。
   - 立即发布：保存后对学生开放。
6. 点击“确定”。

建议先保存为草稿，完成组卷检查后再发布。

#### 3.2.2 组卷

1. 在试卷列表中点击目标试卷的“组卷”。
2. 在弹窗中筛选题型、分类或关键字。
3. 勾选要加入试卷的题目。
4. 顶部会显示已选题数和合计分值。
5. 点击“保存组卷”。

说明：重新保存组卷会以当前勾选结果为准，旧题目绑定会被替换。

#### 3.2.3 发布试卷

1. 确认试卷至少包含一道题目。
2. 在试卷列表中点击“发布”。
3. 二次确认后，学生即可在“在线考试”中看到该试卷。

#### 3.2.4 编辑试卷

点击“编辑”可修改标题、时长、及格分和状态。已发布试卷建议谨慎修改，避免影响学生考试体验。

#### 3.2.5 删除试卷

点击“删除”并确认后，系统会清理该试卷相关的组卷关系、考试记录、答题记录和成绩记录。

### 3.3 成绩管理

菜单路径：左侧导航 -> 成绩管理

#### 3.3.1 查看全部成绩

管理员和教师可以查看所有学生成绩。列表包含学生、试卷、得分、总分、及格状态、提交时间等信息。

可按试卷筛选成绩记录。

#### 3.3.2 成绩排行

点击“成绩排行”可查看成绩排名。可选择某张试卷查看该试卷排行，也可不选择试卷查看全部排行。

#### 3.3.3 查看答题详情

点击成绩记录右侧“查看详情”，可以看到：

- 题型
- 题目内容
- 学生答案
- 正确答案
- 每题得分
- 是否正确
- 答案解析

### 3.4 数据分析

菜单路径：左侧导航 -> 数据分析

数据分析页展示以下信息：

| 指标 | 说明 |
| --- | --- |
| 参考人数 | 当前筛选范围内的成绩记录数量 |
| 及格率 | 及格人数占比 |
| 平均分 | 当前成绩平均值 |
| 最高分 / 最低分 | 当前成绩区间 |
| 成绩分布 | 按分数段统计人数 |
| 及格率统计 | 及格与不及格占比 |
| TOP 10 排行 | 成绩靠前的记录 |

可以通过顶部试卷下拉框筛选统计范围。

## 4. 学生操作指南

以下功能需要使用学生账号登录。

### 4.1 在线考试

菜单路径：左侧导航 -> 在线考试

#### 4.1.1 查看可参加考试

页面以卡片展示已发布试卷。每张卡片显示试卷标题、考试时长、总分、及格分和题目数量。

#### 4.1.2 开始考试

1. 点击试卷卡片上的“开始考试”。
2. 阅读确认提示。
3. 点击确认后进入答题页面。
4. 系统开始倒计时。

注意：同一学生对同一试卷通常只能参加一次。已经提交或超时的考试不能重复开始。

#### 4.1.3 答题页面说明

答题页主要包含：

- 顶部：试卷标题、倒计时。
- 左侧：题号导航，显示当前题和已答题状态。
- 中间：题目内容、选项或判断按钮。
- 底部：上一题、下一题、提交按钮。

#### 4.1.4 作答方式

| 题型 | 操作方式 |
| --- | --- |
| 单选题 | 选择一个选项 |
| 多选题 | 勾选一个或多个选项 |
| 判断题 | 选择正确或错误 |

系统会把答案保存到浏览器本地缓存。刷新页面后，已填写答案通常可以恢复。最终成绩以提交时发送到后端的答案为准。

#### 4.1.5 提交试卷

可以通过以下方式提交：

- 手动点击“提交”或“交卷”。
- 倒计时结束后自动提交。

提交后系统会显示得分结果，可点击“查看成绩”进入成绩页面。

### 4.2 查看成绩

菜单路径：左侧导航 -> 成绩管理

学生只能查看自己的成绩记录。成绩列表显示试卷名称、得分、总分、是否及格和提交时间。

点击“查看详情”可以查看每道题的作答情况、正确答案和解析。

## 5. 常见问题

### Q1：登录提示密码错误怎么办？

确认账号密码是否为 `123456`。如果数据库较早导入过初始化数据，可能需要更新密码哈希。可重新导入 `data.sql`，或联系管理员执行 README 中的密码修复 SQL。

### Q2：登录后看不到某些菜单怎么办？

不同角色菜单不同：

- 学生只能看到工作台、在线考试、成绩管理。
- 教师和管理员可以看到题库、试卷、成绩、统计等管理菜单。

如果角色不正确，请检查数据库 `user.role` 字段。

### Q3：管理员或教师提示“无权限”怎么办？

请退出后重新登录，刷新 Token。若仍然异常，检查数据库角色是否为 `ADMIN` 或 `TEACHER`。

### Q4：考试时刷新页面会丢答案吗？

系统会把答题过程缓存到浏览器本地，刷新后一般可以恢复。建议考试过程中不要清理浏览器缓存，也不要更换浏览器或设备。

### Q5：为什么交卷后是 0 分？

可能原因：

- 未选择答案。
- 多选题答案与正确答案不完全一致。
- 判断题选择错误。
- 考试超时后提交。

可在成绩详情中逐题核对。

### Q6：查看成绩详情提示缺少答题记录怎么办？

说明该成绩记录没有关联到有效考试记录，通常来自旧数据或手工插入数据。新提交的考试会自动生成关联记录。

### Q7：前端页面打不开怎么办？

确认前端服务已启动，并访问 `http://localhost:5173`。如果端口被占用，Vite 会提示新的端口，以终端输出为准。

### Q8：接口文档在哪里？

后端启动后访问：

```text
http://localhost:8080/doc.html
```

## 6. 推荐演示流程

如果用于课程答辩或项目展示，推荐按以下顺序演示：

1. 使用 `admin / 123456` 登录，展示工作台和整体界面。
2. 进入题库管理，新增一道题目或展示筛选功能。
3. 进入试卷管理，创建试卷并组卷。
4. 发布试卷。
5. 使用 `student1 / 123456` 登录，参加刚发布的考试。
6. 提交试卷并查看成绩。
7. 切回管理员或教师账号，查看成绩管理和数据分析。

## 8. 注意事项

- 发布试卷前必须完成组卷，否则学生无法正常考试。
- 删除题目、试卷、分类前请确认影响范围。
- 考试期间不要关闭后端服务，否则提交会失败。
- 前端本地缓存只保存在当前浏览器，不会跨设备同步。
- 若修改了后端端口，需要同步修改前端代理配置。
